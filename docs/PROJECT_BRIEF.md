# Project Brief

## 1. Название проекта

CardMark Sticker Generator.

## 2. Проблема

Игральные карты и карты Таро сложно надёжно распознавать по фотографии без нейросетей: изображения отличаются стилем, освещением, износом, протекторами и углом съёмки. Проект решает это через маленькие машинно-читаемые метки, которые можно наклеить на карты или протекторы.

## 3. Цель

Создать статический генератор печатных листов стикеров CardMark и JSON-манифеста, чтобы в будущем отдельное приложение могло распознавать карты по фото без ИИ, OCR, внешних API и серверной обработки.

## 4. Текущий статус

Проект находится на стадии раннего прототипа. Draft PR #1 содержит первый генератор стикеров, базовый формат CardMark v0 и документацию для управляемого развития.

## 5. Что входит в текущий прототип

- Статическое приложение без сборки.
- Русскоязычный интерфейс.
- Пресеты Таро 78, игральных карт 36 и 54.
- Кастомный список карт в формате `ID;Группа;Название`.
- Предпросмотр печатного листа.
- CardMark v0 как детерминированная SVG-метка 7×7.
- Экспорт SVG, JSON-манифеста, optional PDF и печать через браузер.
- Базовые проектные документы и журнал решений.

## 6. Что не входит в текущий прототип

- Распознавание с камеры.
- Поиск меток на фотографии.
- Определение `markerId` по изображению.
- Определение прямого или перевёрнутого положения карты.
- Серверная часть, API, база данных или облачная обработка.
- Использование ИИ, нейросетей или OCR.

## 7. Целевая аудитория

- Пользователи игральных карт и Таро, которым нужно подготовить колоду для будущего машинного распознавания.
- Разработчики будущего приложения распознавания.
- Участники проекта, которым нужна фиксированная базовая документация и понятный процесс изменений.

## 8. Основной пользовательский сценарий

1. Пользователь выбирает пресет колоды или вставляет кастомный список.
2. Настраивает размер меток, лист, поля и подписи.
3. Печатает лист стикеров или сохраняет его в PDF.
4. Берёт наклейки по порядку и клеит их на карты или протекторы.
5. Отмечает чекбокс на подложке после каждой наклейки.
6. Сохраняет JSON-манифест для будущего приложения распознавания.

## 9. Ограничения

- Проект должен работать статически, без сервера.
- Приложение должно быть совместимо с GitHub Pages.
- Интерфейс приложения должен быть на русском языке.
- CardMark v0 поддерживает только `markerId` от `0` до `127`.
- PDF-экспорт через CDN является optional; печать через браузер обязательна как fallback.
- Для Таро рекомендуется клеить метки на протекторы, а не на сами карты.

## 10. Критерии успеха MVP

- Приложение открывается локально через `index.html`.
- Приложение может быть опубликовано через GitHub Pages.
- Пользователь может выбрать Таро 78, игральные 36 или 54 и увидеть печатные метки.
- Пользователь может создать кастомный набор.
- Пользователь может экспортировать SVG, JSON и использовать PDF/печать.
- Формат CardMark v0 описан достаточно, чтобы начать проектирование будущего распознавания.
- Проект имеет README, CHANGELOG, AGENTS и документы в `docs/` для управляемой разработки.
