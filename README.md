# CardMark Sticker Generator

CardMark Sticker Generator — рабочий статический генератор печатных листов стикеров CardMark. Стикеры можно наклеивать на игральные карты, карты Таро или протекторы, чтобы в будущем отдельное приложение могло распознавать карты по машинно-читаемым меткам без ИИ, OCR, внешних API и серверной обработки.

Опубликованная версия доступна на GitHub Pages:

https://kostgame.github.io/cardmark-sticker-generator/

Этот репозиторий отвечает только за генератор стикеров, печатные материалы и JSON-манифест. Распознавание карт с камеры вынесено в будущий отдельный проект `card-reader`.

## Возможности v0.1.0

- Пресеты: Таро 78, игральная колода 36, игральная колода 54.
- Кастомный набор через строки `ID;Группа;Название`.
- Размеры меток 6, 8, 10, 12 мм и пользовательский размер.
- Цветовые режимы: чёрная метка на светлом фоне, белая метка на тёмном фоне, экономичная печать.
- Форматы листа: A4 и Letter, portrait и landscape.
- Настраиваемые поля, зазоры, плотность, размеры ячейки, припуск резки, секции, запасные и тестовые метки.
- Экспорт SVG, JSON-манифест, optional PDF через CDN `jsPDF` и fallback через печать браузера.
- Публикация как статическое приложение на GitHub Pages без build step.

## Режимы печатного макета

- `Контрольный односторонний` — читаемый лист для проверки набора и ручного контроля.
- `Компактный односторонний` — плотная раскладка с короткими подписями рядом или под меткой.
- `Двусторонний` — лицевая сторона содержит метки, контуры, линии реза и registration marks; оборотная сторона содержит подписи на соответствующих позициях.
- `Домашняя резка ножницами` — увеличенный припуск, заметные пунктирные линии и более удобная ручная резка.
- `Плоттерная резка` — плотная раскладка, registration marks, cut lines, safe area и SVG-слои для дальнейшей обработки.

Для реальной печати сначала сделайте тест на обычной бумаге. В диалоге печати используйте масштаб 100% и отключайте `fit to page` или другое автоматическое масштабирование.

## Локальный запуск

Откройте `index.html` в браузере. Сервер и сборка не нужны.

Для минимальной проверки проекта:

```bash
npm run check
```

На Windows также можно использовать:

```powershell
npm.cmd run check
```

Подробный чеклист лежит в [docs/CHECKS.md](docs/CHECKS.md).

## GitHub Pages

1. Откройте настройки репозитория на GitHub.
2. Перейдите в **Settings → Pages**.
3. В Source выберите **Deploy from a branch**.
4. В Branch выберите `main`.
5. В Folder выберите `/root`.
6. Нажмите **Save**.
7. Откройте опубликованный URL: https://kostgame.github.io/cardmark-sticker-generator/

Ссылки на CSS и JS содержат cache-busting query string, чтобы после публикации браузер запрашивал свежие статические файлы.

## Как пользоваться генератором

1. Разложите колоду по порядку.
2. Выберите нужный пресет или вставьте кастомный список.
3. Настройте размер метки, формат листа, поля, режим печати и подписи.
4. Распечатайте лист стикеров или сохраните его в PDF.
5. Берите наклейки по порядку.
6. Клейте метку в верхний левый угол лицевой стороны карты.
7. Стрелка «верх» должна смотреть к верхнему краю карты.
8. После наклейки отметьте чекбокс на листе или на оборотной стороне подложки.
9. Для Таро лучше использовать протекторы, чтобы не портить карты.

Сама наклейка содержит только машинную метку и маленький ориентир верха. Название карты, ID, группа, стрелка и чекбокс печатаются на подложке.

## Печать и PDF

Самый надёжный способ получить PDF — кнопка `Печать / PDF` и системный диалог браузера. Так лучше сохраняются реальные размеры в миллиметрах.

Кнопка `Скачать PDF` использует `jsPDF` из CDN, если библиотека доступна. Если CDN недоступен, приложение показывает подсказку и предлагает fallback через печать браузера.

Для плоттерной резки может потребоваться импорт SVG в векторный редактор или ПО плоттера, чтобы настроить слои, линии реза и регистрацию под конкретное оборудование.

## Экспорт SVG и JSON

- `Скачать SVG` сохраняет векторный печатный лист в выбранном режиме.
- В плоттерном режиме SVG содержит слои `layer-markers`, `layer-cut-lines`, `layer-backing-labels`, `layer-registration-marks`, `layer-page-labels`.
- `Скачать JSON` сохраняет манифест колоды для будущего приложения распознавания.
- JSON связывает `markerId` с названием карты, группой и типом колоды.

Пример элемента манифеста:

```json
{
  "id": 0,
  "group": "Старшие арканы",
  "name": "Шут",
  "markerId": 0
}
```

## Формат CardMark

Текущая версия использует CardMark v0: собственную квадратную SVG-метку 7×7 с `markerId` в диапазоне `0..127`, ориентационными ячейками и контрольными битами. Подробная спецификация лежит в [docs/CARDMARK_SPEC.md](docs/CARDMARK_SPEC.md).

CardMark v0 остаётся прототипным контрактом. Он достаточен для генератора v0.1.0 и будущих экспериментов с распознаванием, но может измениться до стабильной версии v1.

## Текущие ограничения

- CardMark v0 поддерживает максимум 128 ID.
- Распознавание с камеры не реализовано в этом репозитории.
- PDF-экспорт через CDN является дополнительным; для точной печати рекомендуется browser print.
- Реальные размеры и двустороннюю регистрацию нужно проверять на конкретном принтере и бумаге.
- Для плоттеров может потребоваться дополнительная настройка SVG в специализированном ПО.

## Будущее приложение распознавания

Будущее приложение `card-reader` будет отдельным проектом. Оно сможет:

- читать камеру в браузере;
- искать CardMark-метки на кадре;
- определять `markerId`;
- определять прямое или перевёрнутое положение;
- сопоставлять ID с JSON-манифестом;
- позволять ручную коррекцию результата.
