# CardMark Sticker Generator

CardMark Sticker Generator — статическое одностраничное приложение для генерации печатных листов стикеров CardMark. Метки предназначены для игральных карт и карт Таро, чтобы будущее браузерное приложение могло распознавать карту по фото без ИИ, OCR, внешних API и серверной обработки.

Приложение работает прямо из `index.html`, подходит для GitHub Pages и не требует сборки.

## Возможности

- Пресеты: Таро 78, игральная колода 36, игральная колода 54.
- Кастомный набор через строки `ID;Группа;Название`.
- Размеры меток 6, 8, 10, 12 мм и пользовательский размер.
- Цветовые режимы: чёрная метка на светлом фоне, белая метка на тёмном фоне, экономичная печать.
- Форматы листа: A4 и Letter, portrait и landscape.
- Настраиваемые поля, расстояние между стикерами, секции, запасные и тестовые метки.
- Экспорт SVG, JSON-манифеста, PDF через optional CDN-библиотеку и печать через браузер.

## Локальный запуск

Откройте файл `index.html` в браузере. Сервер и сборка не нужны.

Если браузер блокирует CDN для PDF-экспорта, остальные функции продолжают работать. Для PDF используйте кнопку «Печать / PDF» и сохранение через системный диалог печати.

## GitHub Pages

1. Откройте настройки репозитория на GitHub.
2. Перейдите в **Settings → Pages**.
3. В Source выберите ветку `main` и папку `/root`.
4. После публикации приложение будет доступно по адресу:
   `https://kostgame.github.io/cardmark-sticker-generator/`.

## Как пользоваться генератором

1. Разложите колоду по порядку.
2. Выберите нужный пресет.
3. Настройте размер метки, формат листа, поля и подписи.
4. Распечатайте лист стикеров.
5. Берите наклейки по порядку.
6. Клейте метку в верхний левый угол лицевой стороны карты.
7. Стрелка «верх» должна смотреть к верхнему краю карты.
8. После наклейки отметьте чекбокс на листе.
9. Для Таро лучше использовать протекторы, чтобы не портить карты.

Сама наклейка содержит только машинную метку и маленький ориентир верха. Название карты, ID, группа, стрелка и чекбокс печатаются рядом на бумажной подложке.

## Печать и PDF

Основной надёжный способ получить PDF — кнопка «Печать / PDF» и системный диалог браузера. Так лучше сохраняются реальные размеры в миллиметрах и кириллица.

Кнопка «Скачать PDF» использует `jsPDF` из CDN, если библиотека доступна. Если CDN недоступен, приложение покажет подсказку и откроет печать через браузер.

При печати отключайте масштабирование в драйвере принтера, если нужна точная геометрия меток.

## Формат CardMark v0

CardMark v0 — простой собственный формат метки без внешних библиотек.

- Метка — квадратная SVG-сетка 7×7.
- Поддерживаемые ID: `0..127`.
- Внешняя рамка всегда заполнена и предназначена для будущего поиска метки на изображении.
- Внутренние угловые якоря задают ориентацию: три угла тёмные, нижний правый светлый.
- Оставшаяся 21 внутренняя ячейка кодирует payload.
- Payload: 7 бит ID, 4 бита CRC, 7 инвертированных бит ID, 3 бита версии.
- Версия v0 кодируется битами `000`.
- Один и тот же ID всегда генерирует одну и ту же логическую матрицу.
- Цветовой режим меняет только рендеринг, но не структуру метки.

Публичная функция:

```js
CardMark.generateCardMark(id, {
  sizeMm: 8,
  colorMode: 'dark-on-light'
});
```

Распознавание в этом репозитории не реализовано. Формат описан заранее, чтобы второе приложение могло искать рамку, определять ориентацию, читать payload, проверять CRC и получать ID карты.

## JSON-манифест

JSON-манифест связывает ID метки с картой:

```json
{
  "format": "CardMark",
  "version": "0.1",
  "deckType": "tarot78",
  "markerSizeMm": 8,
  "cards": [
    {
      "id": 0,
      "group": "Старшие арканы",
      "name": "Шут",
      "markerId": 0
    }
  ]
}
```

Этот файл можно будет загрузить во второе приложение распознавания, чтобы сопоставлять найденный `markerId` с названием карты.

## Текущие ограничения

- CardMark v0 поддерживает максимум 128 ID.
- Распознавание с камеры пока не реализовано.
- PDF-экспорт через CDN является дополнительным; для точной печати рекомендуется браузерная печать.
- Метки следует тестировать на конкретном принтере и бумаге, особенно при размере 6 мм.

## Будущее приложение распознавания
